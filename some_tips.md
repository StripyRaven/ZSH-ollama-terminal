# 🛠️ Практическая организация компиляции в проекте

Проект использует `Cargo workspace`, который идеально подходит для управления сложной,
поэтапной сборкой.

Компиляция отдельных крейтов: Вы можете компилировать каждый крейт в workspace
по отдельности для проверки.

```bash
# Пример компиляции только ядра системы
cargo build --package ai-core
# Пример компиляции только веб-интерфейса
cargo build --package web-ui
```

Компиляция конкретных бинарных приложений: Для сборки только CLI или только демона.

```bash
# Собрать только CLI приложение
cargo build --bin zsh-ai-cli
# Собрать только демон
cargo build --bin zsh-ai-daemon
```

Использование фич для контроля сборки:
Cargo features позволяют включать/выключать части системы.

```bash
# Собрать только базовый функционал без веб-интерфейса
cargo build --no-default-features --features "cli,secure-mode"
# Собрать всё, включая экспериментальные функции
cargo build --all-features
```
